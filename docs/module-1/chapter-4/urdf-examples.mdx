# Basic Humanoid URDF Examples

## Learning Objectives
- Analyze complete URDF files for humanoid robots
- Understand how different body parts are represented in URDF
- Identify the connections between torso, limbs, and joints
- Recognize common patterns in humanoid robot URDF structure

## Simplified URDF Example

For learning purposes, here's a more minimal example focusing on the core structure:

```xml
<?xml version="1.0"?>
<robot name="simple_humanoid">
  <!-- Torso -->
  <link name="torso">
    <visual>
      <geometry>
        <box size="0.3 0.2 0.5"/>
      </geometry>
      <material name="grey">
        <color rgba="0.5 0.5 0.5 1.0"/>
      </material>
    </visual>
    <inertial>
      <mass value="10.0"/>
      <inertia ixx="0.5" ixy="0.0" ixz="0.0" iyy="0.6" iyz="0.0" izz="0.3"/>
    </inertial>
  </link>

  <!-- Head -->
  <link name="head">
    <visual>
      <geometry>
        <sphere radius="0.1"/>
      </geometry>
      <material name="grey">
        <color rgba="0.5 0.5 0.5 1.0"/>
      </material>
    </visual>
    <inertial>
      <mass value="2.0"/>
      <inertia ixx="0.02" ixy="0.0" ixz="0.0" iyy="0.02" iyz="0.0" izz="0.02"/>
    </inertial>
  </link>

  <joint name="neck_joint" type="revolute">
    <parent link="torso"/>
    <child link="head"/>
    <origin xyz="0 0 0.5"/>
    <axis xyz="0 1 0"/>
    <limit lower="-0.5" upper="0.5" effort="10" velocity="2"/>
  </joint>

  <!-- Left Arm -->
  <link name="left_upper_arm">
    <visual>
      <geometry>
        <cylinder radius="0.04" length="0.3"/>
      </geometry>
      <material name="grey">
        <color rgba="0.5 0.5 0.5 1.0"/>
      </material>
    </visual>
    <inertial>
      <mass value="1.5"/>
      <inertia ixx="0.02" ixy="0.0" ixz="0.0" iyy="0.02" iyz="0.0" izz="0.002"/>
    </inertial>
  </link>

  <joint name="left_shoulder_joint" type="revolute">
    <parent link="torso"/>
    <child link="left_upper_arm"/>
    <origin xyz="0.2 0 0.3"/>
    <axis xyz="1 0 0"/>
    <limit lower="-1.57" upper="1.57" effort="20" velocity="2"/>
  </joint>
</robot>
```

## Key Structural Patterns

### 1. Hierarchical Structure
- The robot starts with a base link
- Each subsequent link is connected via joints
- This creates a tree structure (no loops)

### 2. Consistent Naming Convention
- Links: Descriptive names (torso, head, left_upper_arm)
- Joints: Indicate parent-child relationship (torso_to_head)

### 3. Symmetry for Limbs
- Left and right limbs follow similar patterns
- Only differ in position and orientation

### 4. Proper Mass Distribution
- Heavier parts (torso) have higher mass values
- Limbs have appropriate mass relative to size

## Independent Validation

After reviewing these examples, you should be able to identify the torso, arms, legs, and joints in a humanoid URDF file. You should recognize the pattern of links connected by joints, and understand how the hierarchical structure represents the physical robot.
