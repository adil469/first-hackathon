# Reading and Interpreting URDF Files

## Learning Objectives
- Systematically read through a URDF file to understand robot structure
- Identify the base link and main body parts
- Trace the kinematic chain from base to end effectors
- Interpret joint constraints and movement ranges
- Understand the relationship between URDF elements and physical robot parts

## Step-by-Step URDF Reading Process

### Step 1: Identify the Robot Definition
Start by finding the root `<robot>` element and noting the robot's name:

```xml
<robot name="my_humanoid_robot">
  <!-- All robot elements go here -->
</robot>
```

This tells you the name of the robot being described and marks the beginning and end of the robot definition.

### Step 2: Locate the Base Link
Find the first link definition, which typically serves as the base of the robot's kinematic structure:

```xml
<link name="base_link">
  <!-- Base link properties -->
</link>
```

The base link is usually the reference frame for the entire robot and often represents the main body or torso.

### Step 3: Map the Kinematic Structure
Follow the joints to understand how links connect:

1. **Find joints** that reference the base link as a parent
2. **Identify the child links** these joints connect to
3. **Trace the chain** from parent to child links
4. **Repeat** for each new link found

```xml
<!-- Joint connects parent to child -->
<joint name="torso_to_head" type="revolute">
  <parent link="torso"/>      <!-- Parent link -->
  <child link="head"/>        <!-- Child link -->
  <origin xyz="0.0 0.0 0.4" rpy="0 0 0"/>  <!-- Position/rotation of joint -->
  <axis xyz="0 1 0"/>         <!-- Movement axis -->
  <limit lower="-1.57" upper="1.57" effort="100" velocity="1"/>  <!-- Movement limits -->
</joint>
```

### Step 4: Understand Link Properties
For each link, examine three key properties:

#### Visual Properties
How the link appears in visualization:
```xml
<visual>
  <origin xyz="0 0 0" rpy="0 0 0"/>  <!-- Position and rotation offset -->
  <geometry>
    <box size="0.3 0.2 0.5"/>        <!-- Shape and dimensions -->
  </geometry>
  <material name="gray">
    <color rgba="0.5 0.5 0.5 1.0"/>   <!-- Color (Red, Green, Blue, Alpha) -->
  </material>
</visual>
```

#### Collision Properties
How the link interacts in physics simulation:
```xml
<collision>
  <origin xyz="0 0 0" rpy="0 0 0"/>
  <geometry>
    <box size="0.3 0.2 0.5"/>        <!-- Usually simpler than visual geometry -->
  </geometry>
</collision>
```

#### Inertial Properties
Physical characteristics for simulation:
```xml
<inertial>
  <mass value="5.0"/>                 <!-- Mass in kg -->
  <inertia ixx="1.0" ixy="0.0" ixz="0.0" iyy="1.0" iyz="0.0" izz="1.0"/>  <!-- Inertia tensor -->
</inertial>
```

## Practical Reading Example

Let's walk through reading a portion of a humanoid URDF:

```xml
<robot name="basic_humanoid">
  <link name="base_link">
    <visual>
      <geometry>
        <box size="0.001 0.001 0.001"/>
      </geometry>
    </visual>
    <inertial>
      <mass value="0.0001"/>
      <inertia ixx="0.0001" ixy="0.0" ixz="0.0" iyy="0.0001" iyz="0.0" izz="0.0001"/>
    </inertial>
  </link>

  <link name="torso">
    <visual>
      <origin xyz="0 0 0.25"/>
      <geometry>
        <box size="0.3 0.2 0.5"/>
      </geometry>
      <material name="grey"/>
    </visual>
    <inertial>
      <mass value="10.0"/>
      <inertia ixx="0.5" ixy="0.0" ixz="0.0" iyy="0.6" iyz="0.0" izz="0.3"/>
    </inertial>
  </link>

  <joint name="base_to_torso" type="fixed">
    <parent link="base_link"/>
    <child link="torso"/>
    <origin xyz="0 0 0.25"/>
  </joint>
```

**Reading Process:**
1. **Robot name**: "basic_humanoid"
2. **Base link**: "base_link" - a tiny reference point
3. **Second link**: "torso" - a grey box (0.3×0.2×0.5m) with 10kg mass
4. **Connection**: Fixed joint "base_to_torso" connects base_link to torso at position (0, 0, 0.25)

### Step 5: Identify Joint Types and Constraints
Look at each joint's type and limits:

- **Fixed**: No movement (permanent connection)
- **Revolute**: Rotational movement with limits
- **Continuous**: Rotational movement without limits
- **Prismatic**: Linear movement with limits

```xml
<joint name="elbow_joint" type="revolute">
  <parent link="upper_arm"/>
  <child link="forearm"/>
  <axis xyz="1 0 0"/>        <!-- Rotates around X-axis -->
  <limit lower="-2.0" upper="1.5" effort="30" velocity="1.0"/>  <!-- Limited rotation -->
</joint>
```

### Step 6: Trace Limbs and End Effectors
Follow the chain to identify complete limbs:

```xml
torso → left_shoulder → left_upper_arm → left_forearm → (left hand/end effector)
```

## Common URDF Patterns to Recognize

### 1. Tree Structure
- No loops in the kinematic chain
- Each link (except base) has exactly one parent
- Multiple branches possible (arms, legs, head)

### 2. Symmetry in Humanoid Robots
- Left and right limbs often follow mirror patterns
- Similar joint types and ranges
- Different position offsets

### 3. Material Definitions
- Usually at the top of the file
- Reusable throughout the robot
- Define colors and appearance

### 4. Coordinate Systems
- Right-hand rule: X forward, Y left, Z up
- Joint axes define movement directions
- Origin positions define relative locations

## Troubleshooting Common Reading Issues

### Issue 1: Complex Hierarchies
**Solution**: Draw the structure on paper, noting parent-child relationships.

### Issue 2: Large Files
**Solution**: Focus on one limb at a time, then connect the parts.

### Issue 3: Confusing Joint Names
**Solution**: Look at parent and child links to understand the connection.

### Issue 4: Missing Visuals
**Solution**: Check that all required elements (geometry, material) are present.

## Practice Exercise

Take the following URDF snippet and describe the robot structure:

```xml
<robot name="simple_robot">
  <link name="base">
    <visual>
      <geometry><cylinder radius="0.1" length="0.2"/></geometry>
    </visual>
  </link>

  <link name="arm_base">
    <visual>
      <geometry><box size="0.05 0.05 0.1"/></geometry>
    </visual>
  </link>

  <joint name="base_to_arm" type="revolute">
    <parent link="base"/>
    <child link="arm_base"/>
    <origin xyz="0 0 0.1"/>
    <axis xyz="0 0 1"/>
    <limit lower="-1.57" upper="1.57" effort="10" velocity="1"/>
  </joint>
</robot>
```

Answer: This robot has a cylindrical base with an arm attached via a revolute joint that rotates around the Z-axis with ±90° movement range.
