# Glossary of Terms

## A

### AI Agent
An intelligent software system written in Python that implements artificial intelligence logic for robot control and decision-making.

### API (Application Programming Interface)
A set of rules and protocols that allows different software applications to communicate and interact with each other.

### Architecture
The fundamental structure and design of a system, particularly referring to the organization of components and their relationships in ROS 2.

### Asset
Any static content used in the documentation site, including images, videos, documents, and other media files.

## B

### Best Practice
A technique or methodology that has proven to be effective and efficient in achieving desired outcomes in robotics and software development.

### Bridge
A connection or interface that enables communication between different systems, particularly between Python AI agents and ROS 2 controllers.

## C

### Client
In the context of ROS 2 services, a node that sends requests to a service server and receives responses.

### Collision
In URDF, the geometric representation used for physics simulation and collision detection, separate from visual representation.

### Command Line Interface (CLI)
A text-based interface used to run commands and interact with computer systems, such as Docusaurus CLI.

### Configuration
Settings and parameters that define how a system or application behaves, often stored in files like docusaurus.config.ts.

### Coordinate System
A system for defining positions and orientations in 3D space, typically using X, Y, and Z axes.

### Controller
A ROS 2 node that interfaces with robot hardware to execute commands and control physical movements.

### Cross-Reference
A link or reference from one part of the documentation to another, enabling navigation between related topics.

## D

### Docusaurus
A modern static site generator for documentation, developed by Facebook, used to create this book.

### Documentation
Written materials that explain how to use and understand a system, process, or technology.

### Domain-Specific Language (DSL)
A specialized computer language designed for a particular application domain, such as URDF for robot description.

### Dynamic
Relating to systems that change over time, particularly in the context of physics simulation and robot motion.

## E

### Environment
The setting or context in which a system operates, including both physical environments (like simulation) and software environments (like development environments).

### Error Handling
The process of managing and responding to errors that occur during system operation.

### Execution
The process of running or carrying out a program or command.

## F

### Fixed Joint
In URDF, a joint type that creates a permanent, rigid connection between two links with no movement allowed.

### Framework
A reusable, semi-complete application that can be specialized to produce custom applications, such as the Docusaurus framework.

### Frontmatter
YAML metadata at the beginning of Markdown files that provides information about the document, such as title and description.

## G

### Geometry
In URDF, the shape definition (box, cylinder, sphere, mesh) that describes the physical form of robot components.

### Glossary
A list of terms with their definitions, typically specific to a particular subject or domain.

### Gazebo
A physics-based 3D simulation environment often used with ROS for robot simulation and testing.

## H

### Hardware
The physical components of a robot or computer system, as opposed to software.

### Humanoid Robot
A robot designed with a human-like body structure, typically including a head, torso, two arms, and two legs.

## I

### Inertial Properties
Physical characteristics in URDF that define how a robot component responds to forces and torques, including mass and moment of inertia.

### Interface
A shared boundary across which two or more separate components exchange information.

### Inverse Kinematics
Mathematical process for determining the joint angles required to position a robot's end effector at a desired location.

### Isaac
NVIDIA's robotics platform and simulation environment, often used with ROS for robotics development.

### Iteration
The process of repeating a procedure or cycle to refine and improve results.

## J

### Joint
In URDF, a connection between two links that defines how they can move relative to each other.

### Joint Limits
Constraints that define the minimum and maximum values for joint movement in revolute and prismatic joints.

## K

### Kinematic Chain
A series of rigid bodies (links) connected by joints that define the motion of a robot mechanism.

### Kinematics
The study of motion without considering the forces that cause the motion, particularly relevant to robot movement.

## L

### Learning Objective
A clear, specific statement about what learners should be able to do after completing a lesson or module.

### Link
In URDF, a rigid body part of the robot that represents a single physical component like an arm segment or torso.

### Load Time
The time required for a web page or application to fully load and become interactive.

### Local Development
The process of building, testing, and modifying software on a personal computer before deployment.

## M

### Mass
In URDF inertial properties, the measure of the amount of matter in a robot component, affecting how it responds to forces.

### Material
In URDF, the visual properties including color and appearance of a link's surface.

### MDX
Markdown extension that allows JSX components to be embedded within Markdown documents.

### Middleware
Software that provides common services and capabilities to applications beyond what's offered by the operating system, such as ROS 2.

### Module
A major section of the book focusing on a specific topic or set of related concepts.

### Motor
A physical device that converts electrical energy to mechanical energy to create motion in robots.

## N

### Node
In ROS 2, a process that performs computation and communicates with other nodes in the ROS graph.

### Node.js
A JavaScript runtime built on Chrome's V8 JavaScript engine, used for running JavaScript on the server.

### npm
A package manager for Node.js that provides access to thousands of reusable packages and libraries.

### NVM (Node Version Manager)
A tool that allows you to manage multiple versions of Node.js on your system.

## O

### Object-Oriented Programming (OOP)
A programming paradigm based on the concept of objects, which contain data and code.

### Optimization
The process of improving a system's performance, efficiency, or other quality attributes.

### Origin
In URDF, the reference point and orientation for a link's geometry and transformations.

## P

### Package
In Node.js, a directory containing a package.json file and related code; in ROS, a reusable software module.

### Parameter
A value that can be configured to customize the behavior of a system or component.

### Physics Engine
Software that simulates the behavior of physical objects, including forces, collisions, and motion.

### Prismatic Joint
In URDF, a joint type that allows linear motion along a single axis.

### Publisher
In ROS 2, a node that sends data to a topic that other nodes can subscribe to.

### Python
A high-level, interpreted programming language commonly used in robotics and AI applications.

## Q

### Quality of Service (QoS)
In ROS 2, policies that define how messages are delivered between publishers and subscribers.

### Query
A request for information from a database or search system.

## R

### rclpy
The Python client library for ROS 2, used to create ROS 2 nodes and interact with the ROS graph.

### Reference
A link or citation to another document, source, or location within the documentation.

### Rendering
The process of generating an image from a model by means of computer programs.

### Request-Response
A communication pattern where one node sends a request and another node sends back a response, used in ROS 2 services.

### Revolute Joint
In URDF, a joint type that allows rotational movement around a single axis with limited range.

### Robot
A mechanical or virtual agent that performs tasks automatically or with guidance.

### Robot Operating System (ROS)
A flexible framework for writing robot software, providing services designed for a heterogeneous computer cluster.

### ROS 2
The second generation of the Robot Operating System, designed for production environments.

### RPY
Roll-Pitch-Yaw, a way to represent rotation using three angles around the X, Y, and Z axes.

## S

### Schema
A structured framework or plan that organizes and defines the elements of a system.

### Search
The functionality that allows users to find specific content within the documentation.

### Service
In ROS 2, a synchronous communication pattern where one node sends a request and receives a response.

### Service Client
In ROS 2, a node that sends requests to a service server.

### Service Server
In ROS 2, a node that receives requests from service clients and sends responses.

### Simulation
The imitation of the operation of a real-world process or system over time, particularly for robot testing.

### Singleton Pattern
A design pattern that restricts a class to a single instance and provides a global point of access to it.

### Static Site Generator
A tool that generates HTML files from templates and content files, such as Docusaurus.

### Subscriber
In ROS 2, a node that receives data from a topic published by other nodes.

### Syntax
The set of rules that defines the combinations of symbols that are considered to be correctly structured in a language.

### System Integration
The process of bringing together different subsystems to function as a unified system.

## T

### Template
A pre-designed document or file that serves as a starting point for new documents with similar structure.

### Topic
In ROS 2, a communication channel where nodes publish and subscribe to data streams.

### Transform
A mathematical operation that changes the position, orientation, or scale of an object in 3D space.

### TypeScript
A programming language that builds on JavaScript by adding static type definitions.

## U

### Unified Robot Description Format (URDF)
An XML-based file format for representing robot models, including their structure, joints, and properties.

### Unit Testing
A software testing method by which individual units of source code are tested to determine whether they are fit for use.

### User Story
A description of a feature from an end-user perspective, used in agile development to capture requirements.

### UX (User Experience)
The overall experience of a person using a product such as a website or computer application.

## V

### Validation
The process of checking that something meets specified requirements or standards.

### Variable
A storage location paired with an associated symbolic name, which contains some known or unknown quantity of information.

### Vector
A quantity that has both magnitude and direction, often used in 3D mathematics.

### Version Control
A system that records changes to files over time so that you can recall specific versions later.

### Visual Properties
In URDF, the geometric and material properties that define how a link appears in visualization tools.

## W

### Webpack
A static module bundler for modern JavaScript applications that processes and bundles assets.

### Workflow
A sequence of connected steps that define how work is performed to accomplish a specific goal.

### Workspace
A directory that contains multiple ROS packages and is used for building and organizing projects.

## X

### XML
Extensible Markup Language, a markup language that defines rules for encoding documents in a format that is both human-readable and machine-readable.

### XML Schema
A description of a class of XML documents along with constraints on their content and structure.

## Y

### YAML
YAML Ain't Markup Language, a human-readable data serialization format commonly used for configuration files.

## Z

### Zero Configuration
A system design approach that requires minimal or no setup or configuration to get started.
